## Теоретическая часть
В рамках лабораторной работы были изучены следующие структуры данных:

- **Список (`list`)** — динамический массив с амортизированной вставкой в конец за `O(1)` и доступом по индексу за `O(1)`.  
  Вставка и удаление в начале — `O(n)`.

- **Связный список (`LinkedList`)** — набор узлов (`Node`), где каждый хранит данные и ссылку на следующий элемент.  
  Вставка/удаление в начало — `O(1)`, доступ по индексу — `O(n)`.

- **Стек (`Stack`)** — структура LIFO (*последний пришёл — первый ушёл*).  
  Реализуется через `list`, операции `append()` и `pop()` — `O(1)`.

- **Очередь (`Queue`)** — структура FIFO (*первый пришёл — первый ушёл*).  
  Эффективно реализуется через `collections.deque` (операции `append()` и `popleft()` — `O(1)`).

- **Дек (`Deque`)** — двусторонняя очередь, позволяющая добавлять/удалять элементы с обеих сторон за `O(1)`.

### Ключевые фрагменты кода

**Связный список (вставка в начало):**
```python
def insert_at_start(self, data):
    """Вставка элемента в начало списка. Сложность: O(1)."""
    new_node = Node(data)
    new_node.next = self.head
    self.head = new_node
    if self.tail is None:
        self.tail = new_node
```
**Проверка скобок (стек):**
```python
def is_balanced_brackets(expression: str) -> bool:
    stack = []
    mapping = {')': '(', ']': '[', '}': '{'}
    for char in expression:
        if char in mapping.values():
            stack.append(char)
        elif char in mapping:
            if not stack or stack.pop() != mapping[char]:
                return False
    return not stack
```
**Проверка палиндрома (дек):**
```python
def is_palindrome(sequence: str) -> bool:
    dq = deque(sequence.lower())
    while len(dq) > 1:
        if dq.popleft() != dq.pop():
            return False
    return True
```

## Ответы на контрольные вопросы

1.  list (динамический массив) — вставка в начало O(n) (сдвиг всех элементов), доступ по индексу O(1).
    Связный список — вставка в начало O(1), доступ по индексу O(n) (нужно пройти узлы).

2.  Стек (LIFO): последний добавленный — первый извлечён.
    Примеры: отмена действия (undo), проверка скобок.  
    Очередь (FIFO): первый добавленный — первый извлечён.
    Примеры: очередь печати, обработка задач веб-сервером.

3.  list.pop(0) требует сдвига всех оставшихся элементов на одну позицию влево → O(n).
    deque.popleft() реализован на двусвязном списке — изменение указателей → O(1).

4.  Стек, потому что отмена — это извлечение последнего действия, что точно соответствует LIFO.

5.  Вставка в начало list — O(n) на каждую операцию → итого O(n²) для 1000 вставок.
    Вставка в начало связного списка — O(1) на операцию → итого O(n). Поэтому list сильно медленнее.
